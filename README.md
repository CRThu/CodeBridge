# CodeMerge: 面向大语言模型 (LLM) 的源代码上下文聚合工具

`CodeMerge` 是一个高效的工程代码打包工具，旨在将分散的项目源码合并为结构化的单一文本文件。其核心用途是为 AI 对话（如 Gemini）提供完整、清晰的项目上下文，解决手动复制多个文件导致的效率低下及语境缺失问题。

## 🛠️ 工作原理

该工具通过以下流程对目标工程进行预处理：

1. **策略加载**：读取 `config.json` 获取全局后缀白名单与默认排除项 。

2. **规则解析**：优先解析目标目录下的 `.agentignore` 文件（采用 `gitwildmatch` 语法），若不存在则回退至默认配置 。

3. **结构建模**：扫描目录树并生成 JSON 格式的层级视图，嵌入在输出文件的头部。
4. **流式合并**：按树状顺序读取文件，注入文件路径标识符并合并其内容，自动忽略二进制或非白名单格式文件。

## 📋 仓库文件说明

| 文件名 | 类型 | 核心作用 |
| --- | --- | --- |
| `code_merge.py` | 源码 | 包含 `pathspec` 匹配逻辑、目录树递归构建及文件合并引擎。 |
| `build.bat` | 脚本 | 基于 `uv` 环境调用 `Nuitka` 的极致压缩编译指令，生成单文件 EXE 。 |
| `config.json` | 配置 | 定义默认允许的扩展名（如 `.cpp`, `.py`, `.js` 等）及排除目录。 |
| `.agentignore` | 配置 | 本项目的忽略示例，包含 `.git/`, `.venv/`, `bin/` 等路径 。|

## 🚀 快速开始  

### 直接下载执行 (Windows)  

如果你是 Windows 用户，可以直接下载预编译的单文件版本，无需安装 Python 环境：

👉 **[下载最新 Release 版本 (v2.0)](https://github.com/CRThu/CodeMerge/releases/download/v2.0/code_merge.v2.0.exe)**

### 开发环境运行

推荐使用 `uv` 运行，系统将自动根据脚本头部的元数据配置环境 ：

```powershell
uv run code_merge.py [项目绝对路径]

```

### 生产环境打包 (Windows)

双击执行 `build.bat`。该过程将：

1. 强制使用 Python 3.12 隔离环境 。


2. 通过 Nuitka 插件系统移除 Python 解释器的冗余信息 。


3. 产出独立可执行文件于 `build_out/` 目录 。



## ⚙️ 配置说明

### 忽略优先级

1. **最高优先级**：目标路径下的 `.agentignore` 文件。
2. **次高优先级**：本工具同级目录下的 `config.json` 中的 `default_exclude`。
3. **最低优先级**：`code_merge.py` 脚本内硬编码的 `default_config`。

### 输出规则

为避免循环读取与文件冲突，合并结果将生成在**目标项目目录的同级位置**，命名格式为 `[工程名].merge.txt`。

## ⚠️ 注意事项

* **字符编码**：本工具默认使用 `UTF-8` 读取与写入。对于非 UTF-8 编码的文件，将通过 `errors='ignore'` 策略跳过乱码字符以保证合并任务不中断。
